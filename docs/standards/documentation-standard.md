# Documentation Standard

1. All documentation goes into the `/docs` folder, except for `README.md` files which can be placed in individual directories.
2. The canonical PRD lives at `docs/prd.md` and contains the ultimate source of truth for the product.
3. Code comments must be extensive, including intro structural definitions and function documentation.
4. Agents must read all files in `docs/standards/` at the start of every session.
